# AltLink

Плагин для PaperMC (Minecraft 1.26.2), реализующий привязку альт-аккаунта к основному через подмену UUID при входе (`AsyncPlayerPreLoginEvent`).

## Возможности

- `/link <main_nick> <alt_nick>` — выполняется с основного аккаунта (alt_nick должен быть онлайн). На альт-аккаунт отправляется 6-значный код.
- `/link <code>` — выполняется с альт-аккаунта для подтверждения. После подтверждения связь сохраняется в `links.yml`.
- При следующем входе альт-аккаунта его UUID подменяется на UUID основного аккаунта — сервер и все плагины (права, инвентарь, приваты, статистика) видят это как один и тот же игрок.
- Данные связок хранятся в `plugins/AltLink/links.yml` и переживают перезапуск сервера.

## Сборка

Требуется:
- JDK 25
- Apache Maven 3.9+

```bash
cd altlink
mvn clean package
```

Готовый jar появится в `target/altlink-1.0.0.jar`. Положите его в папку `plugins/` вашего Paper-сервера 1.26.2 и перезапустите сервер.

> Примечание: `pom.xml` ссылается на `io.papermc.paper:paper-api:1.26.2-R0.1-SNAPSHOT` из репозитория `https://repo.papermc.io/repository/maven-public/`. Если PaperMC к моменту сборки уже перешёл на новую схему версионирования (например `26.2.x.build.+`, как это произошло для веток 1.26.x), обновите версию зависимости в `pom.xml` в соответствии с актуальной документацией https://docs.papermc.io/paper/dev/project-setup/.

## Важно про права и совместимость плагинов

Подмена UUID происходит на этапе `AsyncPlayerPreLoginEvent` с приоритетом `LOWEST`, до того как сервер создаёт профиль и загружает playerdata/permissions. Это означает:

- LuckPerms и аналогичные плагины прав будут хранить и проверять права по UUID основного аккаунта.
- Инвентарь, опыт, статистика (advancements, stats) будут общими, так как файл `playerdata/<uuid>.dat` определяется по (уже подменённому) UUID.
- Приваты/защита территории (WorldGuard, GriefPrevention и т.п.), которые тоже опираются на UUID, будут видеть владельцем альт-аккаунта основной UUID.
- Никнейм (имя) игрока остаётся его собственным — подменяется только UUID, поэтому в чате и tab-листе альт будет отображаться под своим именем, но фактически являться тем же "игроком" с точки зрения данных.

## Команды и права

| Право | По умолчанию | Описание |
|---|---|---|
| `altlink.use` | все | Доступ к `/link` |
| `altlink.admin` | op | Зарезервировано для будущего администрирования связей |

## Формат links.yml

```yaml
links:
  <UUID альт-аккаунта>:
    main-uuid: <UUID основного аккаунта>
    main-name: <имя основного аккаунта на момент привязки>
    alt-name: <имя альт-аккаунта на момент привязки>
```
